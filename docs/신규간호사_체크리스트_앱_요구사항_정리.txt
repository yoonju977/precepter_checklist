신규간호사 체크리스트 앱 요구사항 정리

1. 기본 목적
기존 엑셀 양식인 신규간호사 체크리스트 4주/8주 양식을 기반으로, VS Code에서 실행 가능한 웹앱을 만든다.

핵심 방향:
- 입력은 편하게 한다.
- 최종 출력은 기존 엑셀 양식과 동일하게 나온다.

2. 출력 양식 관련 요구
가장 중요한 조건:
- 디자인이 바뀌면 안 된다.
- 최종 출력 양식은 처음 제공한 엑셀 양식과 동일해야 한다.
- 마지막에 출력 가능해야 한다.

따라서 웹 화면 디자인은 편하게 바꿀 수 있지만, 최종 산출물은 원본 양식을 유지해야 한다.

추천 구조:
입력용 웹 화면
→ 원본 엑셀 양식에 값 자동 입력
→ 전자서명 삽입
→ 완성된 엑셀 또는 PDF 출력

3. 4주 / 8주 체크리스트
원본 엑셀에는 4주/8주 체크리스트가 있다.

앱에서는 다음이 가능해야 한다.
- 4주 체크리스트 입력
- 8주 체크리스트 입력
- 각각 저장
- 각각 출력

4. 역할별 평가 구조
평가자는 역할별로 나뉜다.
- 프리셉티
- 프리셉터
- 교육전담
- 수간호사

각 역할마다 볼 수 있는 점수와 입력할 수 있는 점수가 달라야 한다.

5. 프리셉티 요구사항
프리셉티가 자가평가할 때는 자기가 매긴 값만 보여야 한다.

프리셉티 화면에서 보이는 것:
- 문항
- 자가평가 점수
- 자가평가 메모
- 본인 서명

프리셉티 화면에서 보이면 안 되는 것:
- 프리셉터 평가 점수
- 교육전담 평가 점수
- 수간호사 최종 평가 점수

6. 프리셉터 요구사항
프리셉터는 1명일 수도 있고, 2명일 수도 있다.

단, 프리셉터가 2명이어도 같은 문항을 각각 평가하는 구조가 아니다.
즉, 프리셉터1 점수 / 프리셉터2 점수처럼 따로 두는 방식이 아니다.

프리셉터 평가 점수는 문항당 1개만 있으면 된다.

프리셉터 화면에서 보이는 것:
- 문항
- 프리셉티가 자가평가한 점수
- 프리셉티 메모
- 프리셉터 평가 점수 입력
- 프리셉터 메모
- 프리셉터 서명

프리셉터는 프리셉티가 자가평가한 값만 보면 된다.

프리셉터 화면에서 보이면 안 되는 것:
- 다른 프리셉터가 평가한 전체 흐름
- 교육전담 평가 점수
- 수간호사 최종 평가 점수

다만 이미 다른 프리셉터가 평가한 문항은 잠금 처리하거나 수정 불가 처리하는 방향이 적절하다.

7. 프리셉터 변경 관련 요구
문항마다 담당 프리셉터를 미리 지정하는 방식은 비효율적이다.

이유:
- 처음에는 프리셉터가 1명이었다가
- 중간에 부서이동이나 사직으로
- 프리셉터1에서 프리셉터2로 바뀔 수 있기 때문이다.

따라서 특정 문항을 프리셉터1에게, 특정 문항을 프리셉터2에게 배정하는 방식은 맞지 않는다.

최종 설계 방향:
- 문항별 담당 프리셉터를 미리 저장하지 않는다.
- 일반 문항은 프리셉터 역할이 평가한다.
- 실제로 평가하고 서명한 프리셉터 이름과 서명만 기록한다.

예시:
- 1~40번은 김OO 프리셉터가 실제 평가
- 41번 이후는 박OO 프리셉터가 실제 평가

이렇게 결과적으로 기록되는 것이지, 처음부터 문항별 담당자를 고정하지 않는다.

8. 회색 문항 / 교육전담 평가
원본 양식에서 회색으로 표시된 문항은 프리셉터가 평가하지 않는다.
회색 문항은 교육전담이 평가해야 한다.

따라서 문항에는 담당 프리셉터를 저장하는 것이 아니라, 평가 주체 유형만 저장하면 된다.

평가 주체:
- 일반 문항 → 프리셉터 평가
- 회색 문항 → 교육전담 평가

교육전담 화면에서 보이는 것:
- 회색 문항
- 프리셉티 자가평가 점수
- 교육전담 평가 점수 입력
- 교육전담 메모
- 교육전담 서명

교육전담 화면에서 보이면 안 되는 것:
- 프리셉터 평가 점수
- 수간호사 최종 평가 점수

9. 수간호사 최종 평가
수간호사는 최종 평가자이다.

수간호사 화면에서는 문항마다 아래 점수가 모두 보여야 한다.
- 프리셉티 자가평가 점수
- 프리셉터 평가 점수
- 교육전담 평가 점수
- 수간호사 최종 평가 점수

수간호사는 전체 평가 흐름을 확인하고 최종 점수를 입력한다.

수간호사 화면에서 보이는 것:
- 문항
- 프리셉티 자가평가 점수
- 프리셉터 평가 점수와 실제 평가자
- 교육전담 평가 점수와 실제 평가자
- 수간호사 최종 평가 점수 입력
- 최종 메모
- 최종 서명

10. 점수 저장 구조
문항별로 평가값은 이렇게 저장하는 것이 적절하다.

문항 1개
- 프리셉티 자가평가
- 프리셉터 평가
- 교육전담 평가
- 수간호사 최종평가

프리셉터가 2명이어도 프리셉터 평가 칸은 1개이다.

각 평가에는 다음 정보가 저장되어야 한다.
- 점수
- 메모
- 서명자 이름
- 서명 이미지
- 서명일시

11. 전자서명 요구사항
원래 양식에는 서명란이 있다.
이를 손가락으로 하는 정자 전자서명으로 바꾸고 싶다.

전자서명 방식:
- 손가락
- 터치펜
- 마우스

직접 서명 가능해야 한다.

서명 기능 요구:
- 신규간호사 서명
- 프리셉터 서명
- 교육전담 서명
- 수간호사 서명

서명은 한 번 등록하고 필요한 문항 또는 완료 항목에 일괄 적용할 수 있으면 좋다.

12. 일괄 서명
전자서명은 항목마다 일일이 하기보다 일괄 적용이 필요하다.

예상 기능:
- 현재 역할 서명 등록
- 완료 문항에 일괄 서명
- 현재 화면 문항에 일괄 서명
- 서명 삭제 또는 재서명

단, 실제 평가자 기록은 남아야 한다.

예:
프리셉터 평가 완료 문항
→ 실제 서명자: 김OO
→ 서명일시 기록

13. 중간저장 / 불러오기
중간저장과 불러오기가 가능해야 한다.

요구 기능:
- 자동 중간저장
- 중간저장 파일 다운로드
- 중간저장 파일 불러오기

처음 테스트용으로는 브라우저 저장과 JSON 파일 저장/불러오기 방식이 적절하다.

저장 파일에는 다음 정보가 포함되어야 한다.
- 대상자 정보
- 4주/8주 구분
- 문항별 평가 점수
- 메모
- 전자서명 이미지
- 서명자
- 서명일시
- 저장일시

14. 모바일 사용
모바일에서도 가능해야 한다.

모바일에서 가능해야 하는 기능:
- 프리셉티 자가평가 입력
- 프리셉터 평가 입력
- 교육전담 평가 입력
- 수간호사 최종평가 입력
- 손가락 전자서명
- 중간저장
- 불러오기
- 70점 미만 경고 팝업

모바일 화면은 원본 엑셀 양식을 그대로 보여주는 방식보다 카드형 입력 화면이 적절하다.

단, 최종 출력물은 원본 양식과 동일해야 한다.

권장 구조:
모바일/PC 입력 화면: 반응형 카드형
최종 출력: 원본 엑셀 양식 그대로

15. 저장 방식 관련 방향
초기 테스트 버전:
- 브라우저 자동저장
- JSON 파일 다운로드
- JSON 파일 불러오기

실제 운영용 확장 방향:
- 대상자별 링크 생성
- 역할별 접근 코드 또는 로그인
- 서버 자동저장
- 여러 평가자가 각자 모바일/PC로 접속
- 수간호사가 최종 확인 후 출력

서버 저장이 필요한 이유:
- 프리셉티, 프리셉터, 교육전담, 수간호사가 서로 다른 기기에서 입력할 수 있기 때문이다.

16. 70점 미만 경고 팝업
수간호사가 최종 점수를 저장하거나 최종 서명할 때, 최종 점수가 70점 미만이면 경고 팝업이 떠야 한다.

팝업 문구:
최종 점수가 70점 미만입니다.
그래도 해당 점수로 저장하시겠습니까?

버튼:
- 돌아가서 수정하기
- 그래도 저장하기

돌아가서 수정하기:
- 저장하지 않고 평가 화면으로 돌아감
- 점수 수정 가능

그래도 저장하기:
- 현재 점수 그대로 저장
- 필요 시 사유 기록

추가로 70점 미만 저장 사유 입력칸이 있으면 좋다.

17. 출력 관련 최종 방향
출력은 웹 화면을 그대로 프린트하는 방식보다, 원본 엑셀 파일을 템플릿으로 사용하는 방식이 적절하다.

출력 구조:
원본 엑셀 파일을 템플릿으로 고정
→ 웹앱 입력값을 해당 셀에 자동 입력
→ 전자서명 이미지를 서명칸에 삽입
→ 완성본 엑셀 다운로드
→ 필요 시 PDF 저장 또는 종이 출력

출력 조건:
- 원본 디자인 유지
- 기존 서식 유지
- 4주/8주 각각 출력 가능
- 전자서명 포함

18. 테스트용 프로토타입에 반영한 내용
현재 테스트 방향으로 만든 프로토타입에는 다음 개념이 들어갔다.
- PC / 모바일 반응형 화면
- 역할별 화면
- 브라우저 자동 중간저장
- 중간저장 파일 다운로드
- 중간저장 파일 불러오기
- 손가락/마우스 정자 서명
- 수간호사 최종점수 70점 미만 팝업
- 회색 문항 / 일반 문항 전환 테스트

다만 실제 원본 엑셀의 회색 문항 목록과 출력 셀 매핑은 아직 확정 필요하다.

최종 핵심 요약
예슬님이 원하는 앱은 다음과 같이 정의할 수 있다.

신규간호사 4주/8주 체크리스트를
PC와 모바일에서 역할별로 입력하고,
손가락 전자서명과 중간저장을 지원하며,
수간호사가 최종 평가 후
원본 엑셀 양식과 동일한 형태로 출력할 수 있는 시스템

핵심 설계 원칙:
1. 입력 화면과 출력 양식을 분리한다.
2. 출력 양식은 원본 엑셀 디자인을 유지한다.
3. 프리셉티는 자기 평가만 본다.
4. 프리셉터/교육전담은 프리셉티 자가평가만 보고 평가한다.
5. 회색 문항은 교육전담이 평가한다.
6. 수간호사는 모든 평가값을 보고 최종 평가한다.
7. 프리셉터 담당 문항을 미리 고정하지 않는다.
8. 실제 평가한 사람과 서명일시를 기록한다.
9. 70점 미만이면 저장 전 경고 팝업을 띄운다.
10. 모바일에서도 입력과 서명이 가능해야 한다.
