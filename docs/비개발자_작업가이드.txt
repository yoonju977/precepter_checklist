================================================================
신규간호사 체크리스트 — 비개발자 작업 가이드
(Windows + VS Code + AI 코드 수정 환경 기준)
================================================================

이 문서는 개발 경험이 없는 사용자가 이 프로젝트를 PC에 받아서
AI(Claude 등)의 도움으로 코드를 수정하고 테스트하는 방법을 안내합니다.


================================================================
1단계. 프로그램 설치 (처음 한 번만)
================================================================

아래 3가지를 순서대로 설치하세요.

------------------------------------------------------------
[A] Node.js 22 설치
------------------------------------------------------------
주소: https://nodejs.org/en/download

→ "22.x.x LTS" 버튼 클릭
→ Windows Installer (.msi) 다운로드
→ 실행 후 계속 "다음" 클릭하여 설치 완료

※ 반드시 22 버전 이상이어야 합니다 (이 프로젝트의 빌드 도구 요구사항)

설치 확인 방법:
  PowerShell을 열고 (시작 버튼 → "PowerShell" 검색) 아래 입력:

    node -v

  화면에 "v22.x.x" 처럼 22로 시작하면 성공

------------------------------------------------------------
[B] Git 설치
------------------------------------------------------------
주소: https://git-scm.com/download/win

→ 자동으로 다운로드 시작 → 실행
→ 설치 옵션은 전부 기본값으로 "Next" 클릭하여 설치 완료

설치 확인:
  PowerShell에서:

    git --version

  "git version 2.x.x" 처럼 나오면 성공

------------------------------------------------------------
[C] VS Code 설치
------------------------------------------------------------
주소: https://code.visualstudio.com/

→ "Download for Windows" 클릭 → 설치
→ 설치 중 "PATH에 추가" 옵션 체크 확인 (기본값으로 체크되어 있음)


================================================================
2단계. 프로젝트 받기 (git clone)
================================================================

PowerShell을 열고 아래 명령어를 순서대로 복사 붙여넣기 하세요.
(각 줄 입력 후 Enter)

  # 바탕화면으로 이동
  cd $HOME\Desktop

  # 프로젝트 다운로드 (약 10~30초 소요)
  git clone https://github.com/yoonju977/precepter_checklist.git

  # 프로젝트 폴더로 이동
  cd precepter_checklist

  # VS Code로 열기
  code .

→ VS Code가 자동으로 열리고 파일 목록이 보이면 성공


================================================================
3단계. 의존성 설치 (처음 한 번 + 코드 변경 시)
================================================================

VS Code 상단 메뉴 → 터미널 → 새 터미널 (또는 Ctrl + `)

터미널 창에 아래 입력:

  npm install

→ 설치 중 진행 상황이 표시됨 (1~3분 소요)
→ "added XXX packages" 처럼 나오면 완료
→ 경고(warn)는 무시해도 됩니다, 오류(error)가 없으면 정상


================================================================
4단계. 개발 서버 실행 (코드 수정 후 브라우저 테스트)
================================================================

VS Code 터미널에서:

  npm run dev

→ 아래처럼 나오면 성공:

    VITE v8.x.x  ready in 200 ms
    ➜  Local:   http://localhost:5173/precepter_checklist/

브라우저를 열고 아래 주소로 접속:

  http://localhost:5173/precepter_checklist/

※ 코드 파일을 저장(Ctrl+S)하면 브라우저가 자동으로 새로고침됩니다
※ 서버 종료: 터미널에서 Ctrl + C


================================================================
5단계. 빌드 오류 확인 (코드 수정 후)
================================================================

VS Code 터미널에서:

  npm run build

→ 오류 없이 "built in Xs" 로 끝나면 정상
→ 오류가 있으면 빨간 글자로 표시됨 → AI에게 오류 메시지 전체를 복사해서 전달


================================================================
6단계. AI에게 작업 지시하는 방법
================================================================

AI(Claude 등)에게 요청할 때, 아래 첫 메시지를 그대로 복사해서
원하는 작업 내용을 추가한 뒤 전송하세요.

──────────── [AI에게 보내는 첫 메시지 — 여기서부터 복사] ────────────

이 프로젝트는 병원 신규간호사 체크리스트 웹앱입니다.
React 19 + TypeScript 6 + Vite 8 + Tailwind CSS v4 로 구성되어 있습니다.
GitHub Pages에 정적 배포되며, Google Apps Script를 통해 Google Drive에 저장합니다.

[ 핵심 파일 위치 ]
- 화면 컴포넌트: src/components/
  - layout/ : 신규간호사 정보 입력, 역할 선택, 주차 선택, 교육자 정보 화면
  - checklist/ : 체크리스트 목록 화면, 항목 카드, 점수 입력
  - common/ : 공통 모달(저장완료, 최종제출, 미답변, 낮은점수, 서버불러오기)
  - signature/ : 전자서명 패드
- 전역 상태: src/features/checklist/ChecklistContext.tsx
- 체크리스트 문항 데이터: src/data/checklistData.ts (하드코딩됨)
- 엑셀 출력: src/lib/excel/exportExcel.ts
- Google Drive 저장: src/lib/googleDrive/gasClient.ts
- 타입 정의: src/types/

[ 앱 진행 흐름 ]
신규간호사 정보 입력 → 역할 선택 → 주차 선택 → 교육자 정보 입력 → 체크리스트 입력

[ 역할 종류 ]
- preceptee : 신규간호사 (자가평가)
- preceptor : 프리셉터 (preceptor 문항 평가)
- educator : 교육전담 (educator 문항 평가)
- headNurse : 수간호사 (전체 확인 + 최종평가)

[ 주차 종류 ]
- 4week : 158개 문항
- 8week : 82개 문항

[ 요청 내용 ]
(여기에 원하는 작업을 구체적으로 써주세요)

──────────── [여기까지 복사] ────────────

[ 작업 지시 예시 ]

예시 1 (UI 수정):
  "역할 선택 화면에서 각 버튼의 배경색을 연한 크림슨 색으로 바꾸고
   버튼 높이를 좀 더 크게 해주세요."

예시 2 (기능 추가):
  "체크리스트 화면 상단에 현재 역할과 대상자 이름을 더 크게 표시해주세요."

예시 3 (버그 수정):
  "임시저장 후 '계속 작업' 버튼을 누르면 화면이 깜빡이는 문제가 있어요."

3. 브라우저에서 http://localhost:5173/precepter_checklist/ 새로고침
4. 확인 후 문제 있으면 AI에게 다시 요청

7. gitignore로 제외된 파일 안내 (빌드에 영향 없음)
================================================================

이 프로젝트는 git clone 후 npm install만 하면
추가 파일 없이 바로 실행 가능합니다.

아래 파일들은 git에서 제외되어 있지만 빌드에 전혀 영향을 주지 않습니다:

  gas/Code.gs
    → Google Apps Script 서버 코드
    → GAS 서버 URL이 이미 코드 내에 하드코딩되어 있어 필요 없음
    → 서버 기능(Drive 저장/불러오기)은 이미 운영 중인 서버가 처리

  CLAUDE.md / .claude/
    → Claude Code AI 전용 지시 파일, 일반 사용자 불필요

  public/data/*.csv
    → 체크리스트 문항 원본 데이터
    → 이미 src/data/checklistData.ts 파일에 변환되어 내장됨

  docs/기능테스트시나리오.md 등
    → 내부 개발 문서, 앱 실행과 무관

8. 변경사항 GitHub에 올리기 (배포)
================================================================

코드 수정 완료 후 VS Code 터미널에서:

  # 변경된 파일 목록 확인
  git status

  # 모든 변경사항 스테이징
  git add .

  # 커밋 (따옴표 안에 변경 내용 간단 설명)
  git commit -m "버튼 색상 변경"

→ 배포 확인: https://yoonju977.github.io/precepter_checklist/

9. 자주 발생하는 문제
================================================================

Q: npm install 시 오류가 발생해요
A: node -v 입력 후 22로 시작하는 버전인지 확인하세요.
   아니면 Node.js 22를 새로 설치하세요.

Q: 브라우저에서 화면이 안 나와요
A: http://localhost:5173/precepter_checklist/ (뒤에 /precepter_checklist/ 포함)
   주소가 맞는지 확인하세요.

Q: 코드를 수정했는데 브라우저가 안 바뀌어요
A: npm run dev 가 터미널에서 실행 중인지 확인하세요.
   실행 중이면 파일 저장(Ctrl+S) 시 자동 새로고침됩니다.

Q: npm run build 시 오류가 나요
A: 빨간 오류 메시지 전체를 복사해서 AI에게 전달하면 수정해줍니다.

Q: git push 시 "Permission denied" 오류가 나요
A: GitHub 로그인이 필요합니다.
   VS Code 좌측 소스 제어(Ctrl+Shift+G) → GitHub에 로그인 버튼 클릭
   또는 Git Credential Manager를 통해 로그인하세요.

Q: "cannot find module" 오류가 나요
A: npm install 을 다시 실행해보세요.
